Hexo+Git搭建博客大概流程

Hexo+Github搭建属于自己的博客

在学习中，非常需要一个博客，它不仅可以帮你记录你的劳动成果，最重要的是可以帮你记录你的思想，帮你唤醒当时你在想什么。经过摸索学习，生活中、网络中多个大牛们的帮忙，我也自己搭建了属于自己的博客。刚刚学会，来分享一下，与其他人的教程比较，我的这个教程就显得有些小巫见大巫，有很多不能解决的问题，还请谅解。
大概流程：
[TOC]

  1. 搭建 Node.js环境
  2. 搭建 Git 环境
  3. GitHub 注册和配置
  4. 安装配置 Hexo
  5. 关联 Hexo 与 GitHubPages
  6. Hexo 的最简单操作—–建立新的文件并部署

一、搭建 Node.js 环境

在 Node.js 官网：https://nodejs.org/en/ 下载安装包（随意选啦~~）

一路Next 就行。 当然也可以自己选择安装路径，然后Next到底
然后打开命令提示符，依次输入 node -v、npm -v。若出现版本号说明Node.js环境配置成功

二、搭建Git环境

注：此环境用于将本地的网页、文章提交到Github上

在https://git-scm.com/ 里下载安装包 版本自己选就行然后安装

安装完后桌面右键 打开 Git Bash Here，，输入git –version，若出现版本号说明Git环境配置完成

四、安装配置Hexo

这是个什么东西呢？？引用一下：
Hexo 是一个快速、简洁且高效的博客框架，使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。想要跟多的了解，请阅读Hexo官方文档 https://hexo.io/zh-cn/

使用 npm安装 Hexo：
首先，再一个你喜欢的路径建立一个文件夹（名字随意但不能是汉字），在文件夹中右键，打开 Git Bash Here
然后输入

npm install hexo-cli -g

可能会看到一个WARN 但是没有关系，放心用就行了，不会有问题。

最后输入 hexo version 查看Hexo的版本

这时表明你的 hexo 安装完成。接下来就是输入以下命令来初始化 Hexo

hexo init yourname.github.io
使用完这个指令后 你会发现文件夹中多了一个yourname的文件夹，执行后续步骤。

其中的 yourname 改成你自己的用户名

下一步：运行本地Hexo 服务

输入 hexo server 或者hexo s

注：若hexo serve 不能使用，我们可以输入下面这个指令，安装一下 server

npm install hexo -server –save

安装完成后就可以使用了。

然后访问 http://localhost:4000 ，若能正常访问，说明本地博客已经搭建成功。接下来我们要做的就是部署到Github上。

五、关联Github 与 Hexo

我们用SSH keys 让Hexo与 Github 建立联系。

第一步：生成 SSH keys

输入指令 ssh-keygen -t rsa -C “youremail”
此处 “youremail” 填写你用来接受邮件的邮箱，QQ、163啊什么的都可以

输入上述代码后，下面会弹出，输入密码，这个密码是你在提交项目的时候用的，可有可无，想设置密码就输入密码后回车；不想要密码就直接连续回车就行。

第二步：添加刚才生成的SSH keys到你的Github上

因为我们刚才第一步设置的盘是C盘 所以你会在你的C盘C:\Users\里找到一个 .ssh的文件夹，打开它，找到一个叫id_rsa.pub的文件 这就是我们刚才生成的密钥，将这个文件用记事本打开，复制里面的所有内容粘贴到 https://github.com/settings/ssh 中的 new SSH key中，保存即可。

第三步：测试是否设置成功

输入指令 ssh -T git@github.com
会出现一个提示，直接输入yes回车就行，然后就会看到
Hi You’ve successfully …………(不打了，偷下懒）

第四步：配置Git个人信息

从英文提示可以看到，已经链接成功，但是需要完善个人信息
输入下面的指令：
git config –global user.name “yourname”(yourname改为你的用户名）
git config –global user.email “youremail”（youremail改为你的邮箱）
