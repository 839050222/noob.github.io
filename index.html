在调试程序的时候可能会碰到程序有反调试，其调用的函数为IsDebuggerPresent()。该函数是被Kernel32.dll导出的,该函数没有参数,如果当前程序正在被调试的话,返回值为1,没有被调试的话,返回值为0。在这种程序中可能还会碰到PostQuitMessage（）这个函数，这个函数是用来发送一个WM_QUIT消息 来关闭窗口的 也就是说当IsDebuggerPresent()函数检测到程序被调用时可能会调用这个函数来关闭程序。IsDebuggerPresent()这个API其实是读取到了一个特殊位置的字节值，也就是程序开始时EBX所指的区域+2的值。为0 表示未被调试，为1 则表示正在被调试。

《逆向工程核心原理》

在《逆向工程核心原理》这本书中了解到了 栈帧技术
栈帧就是利用EBP（栈帧指针）寄存器访问栈内局部变量、参数、函数返回地址的技术手段。
调用函数时，会先将函数起始地址（ESP）保存到EBP，并维持在函数内部，这样无论ESP怎么变化，以EBP的值为基准能够安全访问到相关函数的局部变量、参数、返回地址。借助战阵技术管理函数调用时，无论函数有多深、多复杂，调用栈都能得到很好的管理与维护。

PE文件格式
PE文件是windows OS 下使用的可执行文件格式。一个.exe文件运行时 如何加载到内存、从何处运行、运行中需要的DLL又哪些、需要多大的栈/堆内存等大量信息，都以结构体形式存储在PE头中。
基本结构
1.从DOS头到节区头是PE头部分。文件中使用偏移（offset），内存中使用VA (Virtual Address）来表示位置，文件加载到内存时节区大小、位置等都会发生变化。文件的内容一般可分为代码（.text）、数据（.data）、资源（.rsrc） 节，分别保存。

2.各节区头定义了各节区在文件或内存中的大小、位置、属性等。文件/内存中节区的起始位置应该在各文件/内存最小单位的倍数位置上，空白区域用NULL填充。
3.VA指进程虚拟内存的绝对地址，RVA指从基准位置开始的相对地址,PE头内部信息大多以RVA 形式存在
RVA+ImageBase=VA

函数调用约定

Calling Convention 既“函数调用约定”，他是对函数调用时如何传递参数的一种约定。函数调用约定有一下三种：

cdecl(c语言默认)

cdecl是C语言使用的方式，调用者main（）函数直接清理其压入栈内的函数参数，
cdecl方式的好处在于，它可以像C语言的printf（）函数一样，
向被调用者传递长度可变的参数。这种长度可变的参数在其他调用约定中很难实现。

stdcall

stdcall方式常用于Win32 API 该方式由被调用者清理栈。
若想在C语言中使用stdcall方式编译源码，只要使用_stdcall关键字即可

stdcall方式的好处在于，被调用者函数内部存在着栈清理代码，
与每次调用函数时都要用ADD ESP,XXX 命令的方式相比，代码尺寸小

fastcall

fastcall方式与stdcall方式基本类似，但是该方式通常会使用寄存器（而非栈内存）去传递那些需要传递给函数的部分参数。
其优点在于可以实现对函数的快速调用（寄存器在CPU内部），但是有时需要额外的系统开销来管理ECX,EDX寄存器。若调用函数前ECX与EDX中存有重要数据，那么使用前就必须先备份
